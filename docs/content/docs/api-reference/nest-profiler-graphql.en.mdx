## createProfilerFieldMiddleware

```ts
function createProfilerFieldMiddleware(
  cls?: ClsService,
  options?: FieldTracingOptions,
): FieldMiddleware;
```

A code-first `@nestjs/graphql` field middleware that times each `resolveField` as a `graphql-field` span in the Timeline trace and nests the DB/HTTP calls it issues under it. Wire it into `GraphQLModule.forRoot` via `buildSchemaOptions.fieldMiddleware`; `cls` is optional (the shared `nestjs-cls` instance is used when omitted). A passthrough when the profiler is off — but graphql-js still calls it per field once registered, so to run nothing on the GraphQL side when disabled, register it conditionally: `fieldMiddleware: profilerEnabled ? [createProfilerFieldMiddleware()] : []` (resolve the flag in `forRootAsync` when it comes from a `.env` file, as `buildSchemaOptions` is read before that file is loaded).

## GraphQLEntrypointData

Payload of the `graphql` entrypoint (`profile.entrypoint.data`). GraphQL runs over HTTP but is its own entrypoint kind, so this extends `HttpRequestData` with a guaranteed `graphql`. Registering `GraphQLCollectorModule` installs the `graphql` type (`GRAPHQL_ENTRYPOINT_TYPE`) via `GRAPHQL_ENTRYPOINT_TYPE_DEF`, giving operations their own list table, detail tab and an **Operation** filter above the GraphQL list.

## Pre-built filters

These are re-exported from `@eleven-labs/nest-profiler-graphql` and consumed via the `ignoreRequest` option of `ProfilerModule`.

### `ignoreGraphQLPlayground`

```ts
const ignoreGraphQLPlayground: ProfilerRequestFilter;
```

Skips `GET /graphql` requests that serve the Apollo Sandbox UI (`Accept: text/html`). These requests are browser page loads and carry no meaningful GraphQL data.

### `ignoreGraphQLIntrospection`

```ts
const ignoreGraphQLIntrospection: ProfilerRequestFilter;
```

Skips GraphQL introspection queries sent by the playground and other tooling to load the schema. Matches:

- `operationName === 'IntrospectionQuery'` (all major clients), or
- a POST body query containing `__schema` or `__type` (anonymous introspection).

### `combineFilters` _(from `@eleven-labs/nest-profiler`)_

```ts
function combineFilters(...filters: ProfilerRequestFilter[]): ProfilerRequestFilter;
```

Combines multiple filter predicates with an OR - the request is skipped when **any** filter returns `true`.

```ts
import { ProfilerModule, combineFilters } from '@eleven-labs/nest-profiler';
import {
  ignoreGraphQLPlayground,
  ignoreGraphQLIntrospection,
} from '@eleven-labs/nest-profiler-graphql';

ProfilerModule.forRoot({
  ignoreRequest: combineFilters(ignoreGraphQLPlayground, ignoreGraphQLIntrospection),
});
```

## Public exports

```ts
import { GraphQLCollectorModule } from '@eleven-labs/nest-profiler-graphql';
import { GraphQLContextAdapter } from '@eleven-labs/nest-profiler-graphql';
import { createProfilerFieldMiddleware } from '@eleven-labs/nest-profiler-graphql';
import {
  ignoreGraphQLPlayground,
  ignoreGraphQLIntrospection,
} from '@eleven-labs/nest-profiler-graphql';

import type {
  GraphQLCollectorModuleOptions,
  FieldTracingOptions,
} from '@eleven-labs/nest-profiler-graphql';
```

Drive the options from `ConfigService` (or any provider) with `forRootAsync()`, gated per environment with `ConditionalModule` (recommended). It resolves option _values_ only — for the `enabled` flag and when to prefer `forRoot`, see [Enabling and disabling the profiler](/docs/packages/nest-profiler/configuration#enabling-and-disabling-the-profiler).

Types used by the filters and adapter (`ProfilerFilterRequest`, `ProfilerRequestFilter`, `IContextAdapter`, `GraphQLInfo`, `PROFILER_REQ_KEY`, `combineFilters`) are exported from `@eleven-labs/nest-profiler`.
