---
title: Axios/HTTP client profiling
description: Capture outgoing HTTP requests made with HttpService and inspect them in the HTTP Client panel.
---

This tutorial shows how to add the Axios collector to profile outgoing HTTP requests in a NestJS application that uses `@nestjs/axios`.

## Prerequisites

- `@eleven-labs/nest-profiler` installed and configured
- `@nestjs/axios` and `axios` installed

## Step 1 — Install the package

```bash
pnpm add @eleven-labs/nest-profiler-axios
```

## Step 2 — Register the modules

<Callout type="warn" title="HttpModule is not re-exported">
  `AxiosCollectorModule` does **not** re-export `HttpModule`. You must import `HttpModule` yourself
  in every feature module that uses `HttpService`. Register `AxiosCollectorModule.forRoot()` in the
  **same module** as your `HttpModule` import.
</Callout>

Register both `HttpModule` and `AxiosCollectorModule` together in your feature module:

```ts title="posts.module.ts"
import { HttpModule } from '@nestjs/axios';
import { Module } from '@nestjs/common';
import { AxiosCollectorModule } from '@eleven-labs/nest-profiler-axios';
import { PostsController } from './posts.controller';
import { PostsService } from './posts.service';

@Module({
  imports: [HttpModule, AxiosCollectorModule.forRoot()],
  controllers: [PostsController],
  providers: [PostsService],
})
export class PostsModule {}
```

If you have multiple feature modules using `HttpService`, add `AxiosCollectorModule.forRoot()` alongside each `HttpModule` import. The collector is idempotent — registering it multiple times is safe.

## Step 3 — Use HttpService in a service

```ts title="posts.service.ts"
import { Injectable } from '@nestjs/common';
import { HttpService } from '@nestjs/axios';
import { firstValueFrom } from 'rxjs';

@Injectable()
export class PostsService {
  constructor(private readonly http: HttpService) {}

  async findAll(): Promise<unknown[]> {
    const { data } = await firstValueFrom(
      this.http.get('https://jsonplaceholder.typicode.com/posts?_limit=5'),
    );
    return data;
  }
}
```

## Step 4 — Test it

Start your application and trigger a request that makes an outgoing HTTP call:

```bash
# Cache miss — triggers an Axios call to the external API
curl -i http://localhost:3000/posts
```

Copy the `X-Debug-Token` from the response headers, open `/_profiler/{token}`, and click the **HTTP Client** tab.

You will see each outgoing request with:

- **Method** — `GET`, `POST`, `PUT`, etc.
- **URL** — full URL including query parameters
- **Status** — HTTP response status code (e.g., `200`, `404`)
- **Duration** — round-trip time in milliseconds

The toolbar badge displays `{n}req` — for example `3req` when three outgoing requests were made during the incoming request.

## Combining with cache profiling

A common pattern is to fetch from an external API and cache the result. With both `CacheCollectorModule` and `AxiosCollectorModule` registered, you can compare the two panels across requests:

```bash
# First call — Cache MISS, Axios call fires → HTTP Client tab shows 1req
curl http://localhost:3000/posts

# Second call — Cache HIT, no Axios call → HTTP Client tab shows 0req
curl http://localhost:3000/posts
```

## How it works

`AxiosCollectorModule` attaches an Axios request/response interceptor to the `HttpService` instance via `axiosRef`. The request interceptor records the start time; the response interceptor records the end time, status code, and final URL. Entries are written directly into the CLS profile for the current request.

Outgoing requests made outside a request context (e.g., in `onModuleInit`) are silently ignored since there is no active CLS profile.
